# Day 3: Multi-Model Streaming CLI Chatbot

A fail-safe, production-pattern command-line interface (CLI) chatbot capable of streaming tokens in real-time. Built as part of a 30-Day Generative AI Mastery Curriculum, this implementation abstracts underlying API connectivity using OpenRouter to guarantee zero-overhead development pipelines.

## Overview

This mini-app demonstrates the core architectural patterns required for real-time Large Language Model (LLM) interaction. Instead of waiting for an entire API text block generation to complete (high Time-to-First-Token latency), it processes the incoming Server-Sent Events (SSE) data stream to output tokens as they are generated by the model.

### Key Architectural Patterns Covered:
* **Token Streaming:** Utilizing `stream=True` via an OpenAI-compatible interface to drastically decrease perceived latency.
* **Fail-Safe Routing:** Leveraging the `openrouter/free` meta-routing path to bypass individual upstream provider rate-limits (`429`) and missing model endpoints (`404`).
* **Environment Isolation:** Abstracting access tokens securely via `python-dotenv` configurations.
